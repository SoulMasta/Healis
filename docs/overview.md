# Обзор системы

## Назначение

Healis — платформа для централизованного доступа студентов и преподавателей к академической информации и учебным материалам. Цель — объединить расписание, материалы, личные рабочие пространства (интерактивные доски), календарь событий и интеграции с университетскими информационными системами в едином удобном интерфейсе.

## Целевая аудитория

- Студенты (основной фокус — удобный доступ к расписанию и материалам, совместная работа на интерактивной доске)
- Преподаватели (публикация материалов, создание событий и объявлений)
- Администраторы (управление интеграциями, мониторинг и администрирование)

## Ключевой функционал

- Просмотр и синхронизация расписания (учёт групп/подгрупп)
- Доступ и управление учебными материалами (загрузка, каталогизация, предпросмотр)
- Интерактивные рабочие пространства (доски) с заметками/стикерами, вложениями и реакциями
- Комментирование и совместная работа в реальном времени (Socket.IO)
- Календарь событий с уведомлениями (тесты, экзамены, дедлайны)
- Аутентификация через университетское SSO / поддержка JWT для внутренних API
- Персонализация интерфейса (тёмная/светлая тема, настройки пользователя)
- Планируемая интеграция с LLM для аналитики содержимого досок

## Компоненты системы

- Клиентское приложение — SPA на React (веб; мобильные клиенты/обёртки по плану)
- Backend API — Node.js + Express (REST API для управления сущностями)
- Real-time слой — Socket.IO для синхронизации досок и уведомлений
- База данных — PostgreSQL (через Sequelize)
- Хранилище файлов — локальное или облачное S3-совместимое (например, Yandex Object Storage) для медиа и вложений
- Интеграционные адаптеры — модуль для подключения SSO и внешних API университетов

## Краткий пользовательский сценарий

1. Пользователь регистрируется/входит через SSO.
2. Клиент запрашивает список рабочих пространств (workspaces) и расписание через REST API.
3. Пользователь открывает доску — клиент подключается к Socket.IO каналу для синхронизации изменений в реальном времени.
4. Изменения (создание стикера, комментарий, реакция) транслируются на сервер и распространяются остальным участникам.
5. Медиа-файлы загружаются через endpoint загрузки и сохраняются в файловом хранилище, в БД хранится ссылка.
